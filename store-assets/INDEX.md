## 🚀 Recommended Workflow

### Phase 1: Setup (30 min)
1. Create developer accounts:
   - [Google Play Console](https://play.google.com/console) ($25 one-time)
   - [App Store Connect](https://appstoreconnect.apple.com) (free with Apple Developer account)
2. Read [STORE-ASSETS-GUIDE.md](STORE-ASSETS-GUIDE.md)
3. Gather metadata (app name, description, keywords)

### Phase 2: Screenshots (1–2 hours)
1. Start app in emulator (1080×1920 for Android, 1242×2688 for iPhone)
2. Take 5–8 screenshots showing key features
3. Add text overlays using Figma/Photoshop (optional but recommended)
4. Save in appropriate `screenshots/` folder
5. Create app preview video for Apple (15–30 sec, optional)

### Phase 3: Upload (30 min)
1. Upload to Google Play Console
2. Upload to App Store Connect
3. Fill in metadata (description, keywords, privacy policy)
4. Submit for review

### Phase 4: Monitor (ongoing)
1. Google Play: Review in 2–4 hours usually
2. Apple: Review in 24–48 hours typically
3. Check for rejections and iterate
4. Monitor download metrics and update as needed

## 💡 Key Insights

### Why These Dimensions?
- **Google Play (512×512):** Displays at ~144×144 in listings; needs high DPI
- **Google Play (1024×500):** Banner stretches full width; text-safe area is center 400px
- **Apple (1024×1024):** iOS scales from this master; universal for all iPhones
- **Screenshots:** Each store's native resolution to avoid blurring

### Why This Folder Structure?
- **Separation:** Icons & content organized by destination store
- **Clarity:** Easy to know what goes where during upload
- **Scalability:** If you need variants (dark mode, localized), folder structure scales
- **Archival:** Future updates stay organized

### Color & Branding
All images use the app's **emerald** (#0E7C5A) and **gold** (#C9A227) theme. This ensures visual consistency across stores and marketing materials.

## ❓ Troubleshooting

**Q: Images look blurry on store?**  
→ Ensure you're using the *exact* dimensions specified. Re-export if needed.

**Q: "Image is not a PNG/JPG"?**  
→ Your file may be corrupted. Re-generate from source or convert using an image tool.

**Q: What if I want to use different screenshots later?**  
→ Both stores allow updating screenshots without a new app version. Just re-upload.

**Q: Can I submit without screenshots?**  
→ Google Play requires minimum 2. Apple requires at least one per supported device. Videos help but aren't required.

**Q: Do I need to create screenshots in multiple languages?**  
→ Yes, if submitting to multiple regions. Screenshots should match the app's language.
